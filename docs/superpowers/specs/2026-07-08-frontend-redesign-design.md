# Dumpit 프론트엔드 리디자인 — 디자인 스펙

- 날짜: 2026-07-08
- 상태: 컨셉 승인됨 (목업 4라운드), 스펙 검토 대기
- 목업: https://claude.ai/code/artifact/6d8cf91b-20d7-4333-b11a-9fe17a887ea0 (4라운드 = 최종 채택안)

## 1. 배경과 목표

Dumpit은 주의집중·우선순위 설정이 어려운 사용자를 위한 생산성 앱이다. 그러나 현재 디자인(강한 원색
레드/오렌지, 픽셀 폰트 전면 사용, 4px 검정 하드 섀도우, 굵은 외곽선)은 시각적 자극이 커서 타겟
사용자의 집중을 오히려 방해하고, 페이지 간 일관성도 부족하며, 전체적으로 유치해 보인다.

**목표**: 레트로 우주라는 기존 정체성(로고·색 DNA·둥근모꼴)은 유지하면서, 자극의 볼륨을 낮추고
타이밍을 재설계한다 — *평소에는 조용하고, 보상 순간에만 화려하게.*

## 2. 스코프

- **포함**: 비주얼 스킨 전면 교체 — 네비게이션(Header), Sidebar, Footer, 모든 모달, 모든 페이지
  (Home, Dashboard, BrainDump, IdeaDump, Routine, Shop, MyPage, Notice, Admin, Privacy, Terms),
  토스트, 배지, 뽀모도로, 원형 타임테이블, 미니캘린더. 라이트/다크 테마 시스템 신규 도입.
- **제외**: 화면 구성(레이아웃 구조)·정보 배치 변경(추후 별도 작업), 로고 리디자인, 백엔드.

## 3. 디자인 원칙

1. **자극의 타이밍**: 작업 화면은 조용하게. 색·모션이 터지는 곳은 완료/보상 순간뿐.
2. **화면마다 주인공 하나**: 타입 스케일 대비로 "지금 볼 것"을 하나만 크게. 나머지는 물러난다.
3. **픽셀 폰트는 크롬에만**: 로고·섹션 라벨·버튼·숫자(시간/코인/진행률)만 둥근모꼴.
   사용자가 읽어야 하는 콘텐츠(할 일 내용, 본문)는 산세리프.
4. **검정 금지**: 외곽선·그림자에 순수 검정 대신 웜 브라운/컬러를 쓴다. 라이트는 컬러 그림자
   (빛이 든 느낌), 다크는 글로우.

## 4. 컬러 토큰

CSS 변수로 정의하고 Tailwind 색상이 변수를 참조한다. 라이트가 기본, `prefers-color-scheme`으로
초기값 결정, 사용자가 토글하면 `<html data-theme="...">` + localStorage로 고정.

| 토큰 | 라이트 | 다크 | 용도 |
|---|---|---|---|
| `--bg` | `#F7EFDF` | `#1F1B2E` | 페이지 배경 (크림 / 딥 플럼) |
| `--card` | `#FFFDF6` | `#2B2442` | 카드·모달·입력 배경 |
| `--fg` | `#33271E` | `#F2E9D8` | 본문 텍스트 |
| `--sub` | `#8C7C66` | `#9D93A8` | 보조 텍스트 |
| `--line` | `#E0D2B6` | `#413966` | 구분선·비활성 보더 |
| `--edge` | `#3A2C21` | `#141021` | 컴포넌트 외곽선 |
| `--chip` | `#F0DFBB` | `#3A3156` | 시간 칩·서브 배경 |
| `--accent` | `#D95F52` | `#F09355` | 주 액센트 (코랄 / 웜 오렌지) |
| `--accent2` | `#C98A2D` | `#E9B44C` | 부 액센트 (머스터드) |
| `--on-accent` | `#FFFBF0` | `#241E14` | 액센트 위 텍스트 |
| `--shadow-hero` | `#EBC0AC` | `#141021` (+글로우) | 히어로 그림자 |
| `--shadow-sm` | `#DCC5A0` | `#141021` | 일반 오프셋 그림자 |

다크 히어로/강조 요소에는 `0 0 28px rgba(240,147,85,.16)` 글로우를 오프셋 그림자에 추가.
시맨틱 컬러(성공/경고/오류)는 액센트와 별도로 두되 같은 채도 톤으로 뮤트해서 정의한다
(구현 시 기존 사용처 조사 후 확정).

## 5. 타이포그래피

- **디스플레이**: `DungGeunMo`(둥근모꼴, 이미 프로젝트에 있음) — 로고, 섹션 라벨, 버튼 텍스트,
  숫자(시간·코인·진행률·통계). Press Start 2P와 Mona12 CDN 폰트는 제거.
- **본문**: `Pretendard Variable`(npm `pretendard` 패키지, 로컬 서빙) — 콘텐츠 전부.
  폴백: `"Malgun Gothic", system-ui, sans-serif`.
- **스케일**: 화면 주인공(히어로 타이틀) 22px/800 + 픽셀 시간 19px. 본문 14–15px,
  보조 12px, 라벨 12px(자간 +5%). `-webkit-font-smoothing: none` 제거(픽셀 폰트에만 국소 적용).

## 6. 컴포넌트 문법 (기존 클래스 대체)

| 기존 | 신규 | 정의 |
|---|---|---|
| `btn-kitschy` | `btn-retro` | `--card`/`--accent` 배경, 1.5px `--edge` 보더, `3px 3px 0 --shadow-sm`, radius 11px. hover: `-1px,-1px` 떠오름+그림자 4px. active: `3px,3px` 이동+그림자 0 (눌리는 촉감). focus-visible 아웃라인 |
| `card-kitschy` | `card-retro` | `--card` 배경, 1.5px `--edge` 보더, radius 16px, `3px 3px 0 --shadow-sm`. 히어로급 카드만 `5px 5px 0 --shadow-hero` |
| `heading-kitschy` | `label-retro` | 둥근모꼴 12px, `--sub` 색, `--accent2` 2px 도트 밑줄 |
| (신규) | `chip-retro` | 시간·메타 칩: `--chip` 배경, radius 6px, 픽셀 폰트 |
| (신규) | 체크박스 | 14px 사각형, 1.5px `--edge`, radius 4px. 완료 시 `--accent` 채움 |
| (신규) | 구분선 | `border-bottom: 1px dashed --line` |

모달: `card-retro` 문법 + 배경 오버레이는 라이트 `rgba(58,44,33,.35)` / 다크 `rgba(10,8,20,.55)`.

## 7. 시그니처 & 모션

- **픽셀 행성 궤도**(대시보드 히어로): box-shadow 픽셀아트 행성(머스터드 3톤) + 점선 궤도 트랙 +
  9초 주기로 도는 위성(액센트 색). 오늘 진행률 `n/m`을 픽셀 폰트로 표기. 재사용 가능한
  `OrbitProgress` 컴포넌트로 제작.
- **다크 모드 별**: 페이지 배경에 픽셀 별 2겹, 한 겹은 4초 주기 반짝임(opacity).
- **버튼 프레스**: 전 버튼 공통 (문법 표 참조).
- **완료 순간**: 할 일 완료 시 픽셀 버스트(작은 사각 파티클 6–8개, 300ms) + 체크박스 채움.
  코인 획득 시 코인 배지 바운스. — 화려함은 여기에만 허용.
- 모든 모션은 `prefers-reduced-motion: reduce` 시 비활성화.

## 8. 테마 시스템 구현 방식

- `index.css`: `:root`(라이트)와 `[data-theme="dark"]`에 토큰 정의.
  초기화 스크립트: localStorage → 없으면 `prefers-color-scheme`.
- `tailwind.config.js`: `colors.primary` 등 기존 이름을 유지하되 값만 `var(--accent)` 형태로
  교체 → 클래스명 변경 없이 상당수 자동 반영. `boxShadow.kitschy*`는 컬러 그림자 값으로 재정의
  후 점진 개명.
- 토글 UI: SettingsModal에 라이트/다크/시스템 3옵션. Header에는 넣지 않는다(크롬 미니멀 유지).

## 9. 적용 순서 (구현 계획의 골격)

1. 토큰·폰트·전역 CSS(테마 시스템 포함) — 이것만으로 전체 색이 바뀜
2. 공통 컴포넌트 문법 클래스 재정의 + 레이아웃 크롬(Header/Sidebar/Footer/Layout)
3. 모달 9종 + 토스트
4. Dashboard(히어로+궤도 시그니처) → BrainDump → IdeaDump → Routine → Shop → MyPage →
   Notice → Home(랜딩) → Admin → Privacy/Terms
5. 완료 모션(픽셀 버스트) 및 마감 QA(두 테마 × 전 페이지 스크린샷 점검)

각 단계는 독립 커밋. 마지막에 Press Start 2P·Mona12 참조 제거 확인.

## 10. 하지 않는 것

- 레이아웃/정보 구조 변경 (사용자가 이미 설계한 화면 구성 유지)
- 로고 파일 교체 (팔레트가 로고와 자연스럽게 이어지도록 뮤트했으므로 그대로 사용)
- 새 기능 추가 (테마 토글 제외)

## 11. 성공 기준

- 두 테마 모두에서 전 페이지가 새 토큰만 사용 (구 색상 하드코딩 0건)
- 픽셀 폰트가 크롬 요소에만 존재하고 콘텐츠 텍스트에는 없음
- 본문 텍스트 대비 WCAG AA 이상 (라이트 `#33271E`/`#F7EFDF` ≈ 12:1, 다크 `#F2E9D8`/`#1F1B2E` ≈ 13:1)
- reduced-motion에서 상시 애니메이션 0개
- 사용자 확인: "차분한데 덤핏답다"
