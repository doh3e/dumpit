# 아이디어 덤프 "제목 + 목록" 계층 인식 개선

날짜: 2026-07-08
상태: 설계 승인됨 (대화), 스펙 리뷰 대기

## 문제

아이디어 덤프에서 사용자가 대제목을 쓰고 그 아래 번호 목록을 적으면, AI 분석이 제목을 최상위(부모) 아이디어로 삼고 목록 항목을 하위 아이디어로 묶어야 한다. 현재는 전부 독립적인 루트 아이디어로 생성된다.

실패 사례 입력:

```
<덤핏 개선안>
1. ~~~~
2. ~~~~
3. ~~~
```

기대: `덤핏 개선안` 루트 1개 + 하위 3개. 실제: 독립 아이디어 3~4개.

## 원인 분석

계층 인프라(AI 응답 `children` 트리, `Idea.parentIdea`, 재귀 저장, 프론트 미리보기 트리)는 이미 전부 구현되어 있다. 문제는 `OpenAiServiceImpl.extractIdeas`의 프롬프트가 이 패턴을 계층으로 인식시키지 못하는 것.

1. HIERARCHY RULES의 중첩 신호 목록에 "제목 줄 + 번호 목록" 패턴이 없다.
2. "do NOT invent a parent grouping" / "A flat list of roots is perfectly fine"이라는 평탄화 지시가 강해서 gpt-4o-mini가 보수적으로 전부 루트 처리한다.
3. few-shot 예시가 구어체 문장 스타일 1개뿐이라 목록형 입력 예시가 없다.
4. `<덤핏 개선안>` 같은 꺾쇠 제목이 `<user_input>` XML 태그 안에 들어가면서 콘텐츠가 아닌 마크업으로 오인될 여지가 있다.

## 해결책

### 1. 프롬프트 강화 (`OpenAiServiceImpl.extractIdeas`)

프롬프트에 다음 4가지를 반영한다:

1. **제목 패턴 신호 추가** — HIERARCHY RULES에 명시: 단독 제목 줄(`<...>`, `[...]`, `#`으로 시작, `:`으로 끝나는 줄, 또는 목록이 바로 뒤따르는 짧은 단독 줄) 다음에 목록 항목(`1.` `1)` `-` `•`)이 이어지면, 제목 줄이 부모(root)가 되고 그 목록 항목들은 전부 그 자식이 된다.
2. **꺾쇠괄호 오해 방지** — `<덤핏 개선안>`처럼 꺾쇠로 감싼 줄은 마크업/태그가 아니라 사용자가 쓴 제목 콘텐츠이므로 버리지 말고 부모 아이디어 제목으로 쓴다(꺾쇠는 제거).
3. **평탄화 규칙과 충돌 해소** — "do NOT invent a parent grouping" 규칙 유지하되, 사용자가 직접 쓴 제목 줄은 invented가 아니므로 반드시 부모로 쓴다는 단서를 덧붙인다.
4. **EXAMPLE 2 추가** — 실패 패턴 그대로 few-shot 예시로: 입력 `"<덤핏 개선안>\n1. 위젯 추가\n2. 다크모드\n3. 알림 개선"` → 출력: `덤핏 개선안` 루트 + children 3개.

### 2. 아이디어 추출 전용 모델 업그레이드

- gpt-4o-mini는 지시 준수력이 약해 프롬프트만으로 한계가 있을 수 있다. 아이디어 추출에만 더 좋은 모델을 쓴다.
- 새 설정 `openai.idea-model` (기본값 `gpt-4.1-mini`)을 추가하고 `extractIdeas`에서만 사용한다. 나머지 AI 기능(우선순위, 일정 추론, 서브태스크, 브레인 덤프)은 기존 `openai.model`(gpt-4o-mini) 유지.
- `callChatApi`에 모델 파라미터 오버로드를 추가한다(기존 호출부는 변경 없음).
- 모델 선택 근거: gpt-4.1-mini는 4o-mini 대비 지시 준수력이 크게 좋으면서 저렴($0.40/$1.60 per 1M tokens vs $0.15/$0.60). gpt-5 계열은 temperature 파라미터 미지원이라 현재 코드(`temperature: 0.3`)와 비호환 — 제외.
- 비용 영향: 추출 1회당 약 $0.0007 → 약 $0.002 (약 3원). 월 1,000회 기준 약 $2. 사용량은 OpenAI 대시보드에서 모델별로 확인하고, 필요하면 `openai.idea-model` 환경변수로 조절한다.
- 사용자 포인트 차감 정책(고정 5점)은 변경 없음 — 모델 변경은 백엔드 원가에만 영향.

## 동작 경계

- 제목 없이 잡생각만 나열 → 지금처럼 평탄한 루트 목록 (기존 동작 유지).
- 제목 블록 + 무관한 딴생각 혼합 → 딴생각은 별도 루트 (기존 EXAMPLE 1이 커버).
- 제목 블록이 여러 개 → 각 제목이 각자의 루트가 되고 각자의 목록을 자식으로 가진다.
- 파싱·sanitize·저장 경로는 변경 없음. 최대 3단계 깊이 제한 유지.

## 검증

1. `./gradlew test` 통과 (기존 테스트 회귀 확인).
2. 수동 테스트 — 앱에서 아래 입력들로 추출 결과 확인:
   - `<덤핏 개선안>` + 번호 목록 (실패했던 원본 패턴)
   - `# 제목` / `제목:` 스타일 변형
   - 제목 없는 평탄한 잡생각 나열 (평탄 유지 확인)
   - 제목 블록 + 무관한 생각 혼합 (분리 확인)
3. (선택) OpenAI API 직접 호출 스크립트로 수정 전/후 프롬프트 A/B 비교.
